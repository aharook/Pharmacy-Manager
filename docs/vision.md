# Vision

## Назва проєкту
Pharmacy Manager

## Проблема
Аптеці потрібно швидко обліковувати склад, продажі та замовлення без ручних помилок у кількості товарів і сумі чеку. Застосунок має допомогти контролювати залишки, оформлювати продажі та підтримувати знижки для клієнтів із карткою.

## Цільові користувачі
- Працівник аптеки на касі
- Завідувач аптеки
- Адміністратор, який веде склад і правила знижок

## Основні сценарії використання
1. Працівник переглядає склад і знаходить препарат за назвою.
2. Працівник створює замовлення, система перевіряє наявність, списує потрібну кількість і рахує суму.
3. Працівник оформлює звичайний продаж без попереднього бронювання товару.
4. Клієнт використовує дисконтну карту, і система застосовує відповідний тип знижки.

## Нефункціональні вимоги
- Читабельний код із чітким поділом на UI, бізнес-логіку та інфраструктуру.
- Тестованість ключових правил: списання складу, розрахунок ціни, знижки.
- Простота розширення: додавання нових типів знижок і нових каналів збереження без переписування ядра.
- Відновлення після помилки: некоректна операція не повинна псувати стан складу.

## Обмеження першої ітерації
- Без графічного інтерфейсу.
- Без реальної бази даних і мережевої інтеграції.
- Без авторизації, ролей і складної звітності.
- Без інтеграції з фіскальним обладнанням.
- Без повного управління терміном придатності та серіями партій.
