# Testing

## Передумови
	- CMake 3.20+
	- C++17 компiлятор
	- GTest через CMake (FetchContent)

## Збирання
```bash
	cmake -S . -B build
	cmake --build build
```

## Запуск тестiв
```bash
	.\build\Debug\pharmacy_tests.exe
```

## Покритi сценарiї
	- Доменна логiка: обчислення суми замовлення, DIRECT/BOOKING правила, штраф за пропущене бронювання.
	- Валідацiя вводу: порожнi назви, дублiкат товару, некоректний тип продажу, нестача залишку, порожнiй `orderId`.
	- Edge cases: нульова кiлькiсть/цiна товару, зменшення залишку до нуля.
	- Iнтеграцiя: створення замовлення з оновленням складу, бронювання, файлове збереження через тимчасовi файли.

## Примiтки
	- Тести створюють тимчасовi файли у робочiй директорiї та видаляють їх пiсля виконання.
